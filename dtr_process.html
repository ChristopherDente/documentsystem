Face Recognition Attendance

A comprehensive guide to deploying and managing the AI-powered Daily Time Record (DTR) system.

1

Environment Setup

Before running the face recognition engine, you must prepare your Python environment. We highly recommend using a virtual environment to prevent dependency conflicts with other system-level packages.

Prerequisites:
  • Python 3.9 or higher
  • CMake (required for building dlib)
  • A working webcam or IP Camera stream
# Clone the repository
git clone https://github.com/christopher/dtr-face-recognition.git
cd dtr-face-recognition

# Create and activate virtual environment
python -m venv venv
source venv/bin/activate # On Windows use: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
2

System Configuration

Once the environment is ready, you must configure the camera source and API endpoints. Locate the config.json file in the root directory.

Property Type Description
camera_source Integer / String Use 0 for local webcam, or provide an RTSP stream URL for IP cameras.
tolerance Float Strictness of face matching. Default is 0.45. Lower means stricter.
api_url String The Laravel backend endpoint where attendance logs are pushed.
3

Employee Registration & Encoding

To recognize employees, the system needs baseline facial encodings. Place clear, front-facing photos of employees inside the /dataset/employees/ folder. The filename should be their unique Employee ID (e.g., EMP-1029.jpg).

After adding the photos, run the encoding script to process and save the facial features into a secure binary file:

python encode_faces.py --dataset ./dataset/employees --encodings encodings.pickle
Note: Re-run this script every time a new employee is added to the system.
4

Live Recognition & Syncing

Start the live attendance tracker. The system will open the camera feed, draw bounding boxes around recognized faces, and log the time-in/time-out events to a local CSV file.

python run_attendance.py

A background cron job (or scheduled task) will parse the attendance_log.csv every 5 minutes, validate the data, and dispatch a POST request to the Laravel backend, ensuring the HR dashboard remains up-to-date in real time.

Architecture

System Architecture

High-level overview of how the Python AI engine communicates with the web infrastructure.

Component Breakdown

Edge Device (Python)

Handles the heavy lifting of facial detection (HOG/CNN algorithms) and encoding comparisons. Runs continuously at the physical office location.

REST API (Laravel)

Acts as the secure gateway. Receives JSON payloads containing employee IDs and timestamps, authenticates via bearer tokens, and writes to the DB.

Database (MySQL)

Centralized storage for employee records, daily time records, overtime calculations, and shift schedules.

Web Client (Vue/Bootstrap)

The HR interface used to generate payroll reports, manage employee data, and view real-time attendance graphs.

Guidelines

Developer Guides

Best practices for maintaining and extending the DTR application.

Coding Standards

  • Python: Follow PEP 8 guidelines. All functions must include comprehensive docstrings detailing expected parameters and return types.
  • Laravel (PHP): Adhere to PSR-12 coding standards. Keep controllers thin; offload complex business logic to Service classes (e.g., AttendanceCalculationService).
  • Security: Never hardcode API keys or database credentials. Always utilize the .env file for configuration management.

Git Workflow

We use a standard Feature Branch Workflow. Create branches off develop using the naming convention feature/issue-number-description. Submit Pull Requests requiring at least one approving review before merging.
